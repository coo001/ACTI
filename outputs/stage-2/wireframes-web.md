# Wireframes (Web, Mobile First) — 연기 스타일 MBTI

> Stage 2.3 산출물 — 5개 화면 ASCII 와이어프레임
> 작성일: 2026-05-14

## 와이어프레임 전제

| 항목 | 값 |
|---|---|
| 1차 설계 폭 | **375px** (iPhone 표준) |
| 지원 폭 범위 | 360~430px (모바일) |
| 최대 컨테이너 | 480px (데스크톱에서도 모바일 폭 유지) |
| 단위 | 모든 박스는 상대적 크기 (실제 px는 Step 2.4 토큰에서 확정) |
| 톤·색 | **이 문서에서는 결정 안 함** (Step 2.4 토큰 정의 후 적용) |
| 텍스트 | 위트 카피 예시 1개만 — 실제 카피는 Phase 2 콘텐츠 작업 |

## 와이어프레임 표기 규칙

```
┌─ ─┐  컨테이너/카드 경계
│   │
└─ ─┘
━━━   강조 경계 (CTA·중요 영역)
···   여백 영역
[TEXT]  배지/태그
(...)  보조 텍스트 (회색·작음)
{...}  동적 콘텐츠 (유형별로 다름)
```

### 요소 인벤토리
| 요소 | 위치 | 비고 |
|---|---|---|
| 서비스명 | 최상단 | 작게, 헤더 느낌 |
| 도메인 시그널 | 헤더 직하 | **가장 큰 시각 비중** ("연영과/배우 너만의 연기 인간 진단") |
| 코드 카드 미리보기 | 시그널 직하 | 1개만 (위트 톤 신호, 호기심 보존) |
| 분량 표시 | CTA 바로 위 | 작게, 이탈 방어 |
| 시작 CTA | 첫 뷰포트 안 | 풀폭, primary 컬러, 화살표 아이콘 |
| 푸터 | 스크롤 영역 | 만든이/피드백 링크 |

### 정보 위계 (Stage 2.1) → 시각 위계 (와이어프레임)
1. 도메인 시그널 → 가장 큰 텍스트
2. 시작 CTA → 가장 큰 버튼
3. 분량 표시 → CTA 옆 작게
4. 서비스명 → 헤더 작게
5. 코드 카드 → 시그널과 CTA 사이 (위트 톤 미리보기)
6. 푸터 → 첫 뷰포트 밖

### 요소 인벤토리
| 요소 | 위치 | 비고 |
|---|---|---|
| 이전 버튼 ← | 좌상단 | 작게, 1번 문항에서는 비활성 |
| 진행바 + 카운터 | 상단 sticky | 진행바 가로 풀폭, 카운터 우측 |
| 시나리오 카피 | 본문 상단 | 1~3줄, 굵지 않게 |
| 선택지 카드 ×4 | 세로 4개 | 각 카드: 아이콘(좌) + 카피(우) |
| (없음) 다음 버튼 | — | 선택 즉시 자동 진행 |

### 진행 상태별 변화
- **1번 문항**: 이전 버튼 비활성 (또는 비표시)
- **마지막 문항(12 / 12)**: 시각 변화 없음. 단, 선택 후 자동으로 결과 페이지로 이동
- **이전 답이 있는 문항**: 해당 선택지 카드 강조 (테두리·체크 마크)

### 인터랙션
- 카드 탭 → 0.2~0.3초 강조 애니메이션 → 자동 다음 문항
- ← 탭 → 이전 문항으로 이동 (답 보존)
- 카드 호버 (데스크톱) → 살짝 들림 + 그림자 강화 (모바일에선 미적용)

### 캡처 영역 1뷰포트 검증
1. 헤더 (~48px)
2. 코드 배지 (~60px)
3. 위트 네이밍 3줄 (~120px)
4. 구분선 + 여백 (~40px)
5. 캐치프레이즈 2줄 (~80px)
6. 캐릭터 묘사 4줄 + 여백 (~140px)
7. 워터마크 (~40px)

**합계 ~528px** → 640px 뷰포트 안 (안전 마진 ~110px)

### 시각 위계 (스크롤 영역)
1. 공유 액션 그룹 → 가장 강한 시각 (큰 카드, primary 색)
2. 천적/베프 카드 → 클릭 어포던스 명확
3. 어울리는 역할/장르 → 텍스트 위주
4. 다시 풀기 → 가장 약한 (secondary, 작은 버튼)

### S3와의 차이점
| 요소 | S3 (본인) | S3' (수신자) |
|---|---|---|
| 안내 텍스트 ("친구가...") | 없음 | 있음 (캡처 영역 상단 작게) |
| 캡처 영역 자체 | 동일 | 동일 |

| 천적/베프/역할 | 동일 | 동일 |

### CTA 시각 강도 비교
- S3'의 "나도 풀어보기" > S3의 공유 액션 > S3의 천적/베프 카드 > 다시 풀기

## 🎯 Step 2.4로 넘기는 것

확정된 구조 위에 **3단계 디자인 토큰**을 입힌다:

1. **Base 토큰** — primitive 색·간격·폰트 크기 (Gray-50, 12px, 16px, …)
2. **Semantic 토큰** — 용도 기반 (background-default, text-primary, border-subtle, …)
3. **Component 토큰** — 컴포넌트별 (button-primary-bg, card-padding, badge-radius, …)

**+ WCAG AA 명암비 검증** (텍스트 4.5:1 이상)
**+ globals.css 산출** (Tailwind/CSS Variables 형태)

비주얼 방향성(파스텔 + 위트 카피)을 토큰 컬러로 구체화:
- Background: 따뜻한 크림 또는 옅은 파우더 컬러
- Primary: 살짝 채도 있는 파스텔 (예: 코랄 핑크·미스티 블루)
- 16유형 또는 4그룹 시그니처 컬러 — 결정 필요 (Step 2.4)
