# Solution Outline — 연기 스타일 MBTI

> Stage 0.3 산출물 — 솔루션을 구성하는 핵심 요소들
> 작성일: 2026-05-13

## 한 줄 솔루션 정의

**4축 16유형 체계 + 위트 있는 별명 + 시나리오 4지선다 12~14문항 + 캡처 한 장에 다 담기는 결과지** — 연기인을 위한 자기진단·공유 놀이거리.

---

## 1. 진단 체계 (4축 16유형)

| # | 축 이름 | 양극 (왼쪽) | 양극 (오른쪽) | 약자 |
|---|---|---|---|---|
| 1 | **접근 철학** | 메소드 (감정이입형) | 테크닉 (외적 계산형) | **M / T** |
| 2 | **현장 운영** | 즉흥 (리바이싱형) | 설계 (술계 준비형) | **I / P** |
| 3 | **준비 사고법** | 직관 (감각으로 진입) | 분석 (구조로 진입) | **N / A** |
| 4 | **표현 통로** | 신체 (외적 표현 중심) | 내면 (정서·심리 중심) | **B / S** |

→ 4글자 조합으로 **16유형** 생성 (예: `MINB`, `TPAS`, `MIAS`, `TINB` …)

### 축 설계 노트

- 축 1·3 미세 분리: 메소드/테크닉(접근 철학)과 직관/분석(준비 단계 사고)이 결이 다르게 작동하도록 설계 — "감정 vs 논리"는 두 축에 겹쳐 보여서 폐기, "직관 vs 분석"으로 교체
- 축 4(신체/내면)는 표현 통로 차원 — 같은 메소드여도 몸으로 표출하는 타입과 안으로 응축하는 타입이 분리됨

---

## 2. 16유형 네이밍 톤

**선택된 톤: 위트 있는 별명형**

- 진지함 30 / 재미 70
- 시적·풍자적·B급 감성
- "야 이거 나야 ㅋㅋㅋ" 반응 유도 — 단톡방 짤맛 최적화
- 자기소개 용도보다는 **친구 사이 별명** 용도

### 네이밍 예시 (확정 톤 샘플)

| 유형 코드 | 네이밍 (예시) |
|---|---|
| MINB | 비 오는 날 뛰쳐나가는 햄릿형 |
| TPAS | 대본에 형광펜 7색 쓰는 인간 |
| MIAS | 새벽 3시에 캐릭터한테 전화하는 타입 |
| TINB | 리허설 안 가도 무대 위에선 천재인 척 |

> 위는 톤 샘플. 16개 네이밍 전체는 Stage 1 콘텐츠 작업에서 확정.

## 3. 결과지 화면 구성

### 캡처 한 장에 들어가야 할 요소 (위에서 아래 순서)

1. **유형 코드** — `[MINB]` 등 4글자
2. **메인 네이밍** — 위트 있는 별명 한 줄
3. **한 줄 캐치프레이즈** — SNS 캡션으로 그대로 쓸 수 있는 펀치라인
   - 예: "감정 한번 잡으면 우산도 안 챙김"
4. **캐릭터 묘사 (3~4줄)** — 위트+공감 톤. 너무 길지 않게.
5. **🎬 어울리는 역할 / 장르** — 사용자가 핵심 가치로 꼽은 가이드
   - 3~4개 (예: 멜로드라마 주인공, 인디 영화 주연, 1인극 모놀로그)
6. **⚔️ 천적 유형 / 🤝 베프 유형** — 다른 유형 1개씩 추천 (비교 재미 트리거)
7. **공유 버튼 + 다시 풀기 버튼**

### 결과지 설계 원칙 (Stage 0.2 원칙과 정합)

- ✅ **캡처 한 장 = 의미 완결** (스크롤 없이 한 화면에 들어와야 함)
- ✅ **친구 결과지와 비교 트리거 내장** (천적/베프 유형)
- ✅ **연기 맥락 살아있음** (역할/장르 추천 = 일반 진단과 차별점)

## 4. 테스트 문항 형식

**선택된 형식: 4지선다 시나리오**

### 구조

- **문항 수**: 12~14문항 (1축당 3~3.5문항)
- **소요 시간**: 2~3분
- **응답 방식**: 한 시나리오 당 4개 선택지, 각 선택지가 한 축의 한 극(極)을 대표

### 문항 예시

```
Q. 리허설 중 상대 배우가 갑자기 다른 톤으로 치고 들어왔다. 나는?

  A) 즉각 그 톤에 맞춰 받아친다           → [즉흥]
  B) 일단 끝까지 가고 끝나고 얘기한다      → [설계]
  C) 그 톤이 왜 나왔는지 감정선부터 짚어본다 → [직관]
  D) '이 장면 비트가 어디서 어긋났는지' 분석한다 → [분석]
```

### 형식 설계 노트

- **연기 전문성 양념 가장 잘 살아남** — 일반 진단 콘텐츠와 결정적 차별점
- **단점**: 시나리오 12~14개 다 직접 써야 함 (제작 난이도 ★★★)
- 시나리오 소재 후보: 리허설, 대본 첫 받았을 때, 공연 직전, 사후 평가, 캐스팅 거절, 슬럼프, 동료와 합 안 맞음 등

## 5. 솔루션 ↔ Stage 0.2 설계 원칙 정합성 체크

| 설계 원칙 | 어떻게 충족하는가 |
|---|---|
| 1. 공유 가능성 > 분석 정확도 | 한 줄 캐치프레이즈 + 캡처 한 장 구조 |
| 2. 캐릭터성 강한 네이밍 | 위트 있는 별명형 톤 채택 |
| 3. 테스트가 너무 길지 않을 것 | 12~14문항, 2~3분 |
| 4. 결과지 비교 재미 | 천적/베프 유형 + 4글자 코드로 같음/다름 즉시 인지 |
| 5. 연기 맥락 살아있을 것 | 시나리오 4지선다 + 역할/장르 추천 |

→ 5개 원칙 모두 솔루션 구성요소에 1:1 매핑됨.

## 6. Stage 0.4로 넘기는 것 (검증할 항목)

- **실현 가능성**: 1인 사이드 프로젝트로 만들기에 적절한 스코프인가? (16유형 콘텐츠 + 12문항 + 결과지 UI = 작업량 추정)
- **차별점 검증**: "MBTI 편승 진단 사이트는 많은데 이건 왜 다른가" 메시지가 5개 설계요소만으로 충분히 전달되나
- **리스크 식별**:
  - 16개 캐릭터 묘사·네이밍 톤 일관성 유지 가능한가
  - 시나리오 12~14개 자체 제작 가능한가 (연기 도메인 지식 깊이)
  - 모바일 결과지 캡처 UX가 정말 한 화면에 들어오는가
- **유사 사례 빠른 점검**: 비슷한 톤·포맷의 진단 사이트가 이미 있는가 (있다면 우리 차별점은?)
