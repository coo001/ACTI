## 3. 리스크 식별 & 대응책

### 🚨 리스크 A — 톤 일관성 붕괴 (가장 큼)

- **내용**: 위트형 네이밍 16개를 같은 결로 유지하며 각 유형 특성 반영이 어려움. 어떤 건 빵 터지고 어떤 건 어색하면 평균값이 깎임.
- **대응책 (사용자 결정 반영)**:
  - AI로 1차 초안 16세트 작성
  - 사용자가 연기 전문성 관점에서 각 유형마다 "이게 정말 그 유형스러운가" + "위트가 살아있나"를 폴리싱
  - 자체 평가 5점 척도로 약한 것 재작업 (≤3점은 폐기/재작성)
- **잔여 리스크**: 사용자가 16개 다 마무리하기 전에 동력 빠질 가능성 → "제작의 재미 자체가 본질" 이라는 가치관으로 완충

### 🚨 리스크 B — 4축 상관관계 문제

- **내용**: 메소드↔직관, 테크닉↔분석이 함께 가는 경향. "직관/분석"으로 미세조정했지만 잔여 상관 가능. 결과적으로 16유형 중 자주 나오는 4~6개 / 거의 안 나오는 10개가 갈릴 수 있음.
- **대응책**:
  - 시나리오 문항 설계 시 의도적으로 "메소드+분석" "테크닉+직관" 같은 비주류 조합도 자연스럽게 잡히도록 배치
  - β 테스트 단계에서 학과 친구 10명 정도에게 풀려보고 분포 확인 → 한쪽으로 쏠리면 문항 재설계

### 🚨 리스크 C — 모바일 결과지 캡처 가독성

- **내용**: 결과지 7개 요소를 모바일 한 화면에 다 넣으면 글자 작아져서 캡처 시 안 읽힘.
- **대응책**:
  - 결과지를 **"캡처 영역"** (유형 코드 + 네이밍 + 캐치프레이즈 + 캐릭터 묘사 3줄) + **"스크롤 영역"** (역할/장르, 천적/베프, 공유 버튼) 로 분리
  - 캡처 버튼은 위쪽 4요소만 자동 캡처해서 다운로드/공유
  - 모바일 first 디자인 (데스크탑은 같은 컴포넌트 가운데 정렬)

## 4. 유사 사례 점검

- "○○인의 MBTI" 포맷 자체는 다수 존재 (작가, 게이머, 술 스타일 등)
- 한국 진단 사이트(포카·동물상·얼굴상) 일부는 매우 인기
- ✅ **"연기 스타일" 도메인은 (적어도 한국어권에서 잘 알려진 건) 비어 있음**
- ✅ 타겟이 명확한 도메인(연극영화과)이라 마케팅 0원이어도 자체 확산 가능

- **30초 검색**: "연기 스타일 MBTI", "배우 MBTI", "연극인 진단" 등 키워드로 동일/유사 서비스 존재 여부 한 번 확인
- 발견 시 → 차별화 포인트 재정의 (예: 우리는 시나리오 4지선다, 위트형 네이밍 등)

## 5. 자체 확산 시나리오 시뮬레이션

### 최선 시나리오 (성공 조건)

1. 사용자 → 학과 친구 1명에게 결과지 전달
2. 친구 "ㅋㅋ 야 너 진짜네" 반응 → 친구도 풀어봄
3. 친구가 단톡방에 캡처 던짐 → 단톡방 5명 추가 유입
4. 그중 1명이 인스타 스토리에 올림 → 타 학과 학생 유입
5. 학과 내 "○○○야" 별명 효과 발생 → 재방문/추가 공유

→ **자체 확산 발동 조건**: 결과지 캡처 한 장에서 "오 너 ○○형이구나"가 1초 안에 인지되어야 함. 위트 네이밍 + 4글자 코드로 충족.

### 최악 시나리오 (실패 조건)

- 결과지 어색함 → 친구에게 안 보냄 → 확산 0
- 시나리오가 연기인스럽지 않음 → "이거 그냥 MBTI잖아" → 차별점 무력화
- 결과 분포가 4~6유형에 쏠림 → 비교 재미 0 → 단톡방 한 번 돌고 끝

## 6. 사용자 결정 반영 사항

| 검증 질문 | 사용자 답변 | 의미 |
|---|---|---|
| 16세트 콘텐츠 자체 작성 가능 여부 | **AI 도움 + 본인 퀄리시(폴리싱)** | 리스크 A를 AI 보조 + 본인 도메인 지식으로 분담 |
| MVP 단순화 (8유형 v1) 권유 | **거절. 16유형 풀 사이즈로 감** | "제작의 재미가 본질" — 사이드 프로젝트 정체성 명확 |

→ **프로젝트 정체성 재확인**: 이건 빠른 출시·완성도 최적화 프로젝트가 아니라, **만드는 과정 자체가 가치인 사이드 프로젝트**. 따라서 스코프 축소보다는 콘텐츠 퀄리티에 시간 투입하는 방향이 맞음.
