# User Stories — 연기 스타일 MBTI

> Stage 1.2 산출물 — Must 11개 + Should 1개 기능에 대한 사용자 스토리
> 작성일: 2026-05-14

## 스토리 작성 원칙

- **포맷**: `[페르소나]로서 / [상황·동기]에서 / [무엇을] 하고 싶다 / [그래서] 어떻게 된다`
- **수용 조건 (AC)**: 이 기능이 "됐다"라고 말할 수 있는 최소 조건 3~5개
- **연결 기능**: feature-priority.md 의 F번호
- **메인 페르소나**: 연극영화과 학생 (20대 초중반, 모바일 사용)
- **1순위 유입 트리거** (Stage 1.2 결정): 학과 커뮤니티/에타에 누군가 "이거 해봄 ㅋㅋㅋ" 글 올림 → 댓글/링크로 유입

---

## 📌 페르소나 약어

- **P1** — 연극영화과 학생 (메인, "민지")
- **P2** — 친구 결과 캡처 보고 따라 들어온 비전공 친구 (확산, "재현")
- **P3** — 다른 학교 연영과 학생 / 신인 배우 (SNS 유입, "수아")

## US-01 — 첫 1초 안에 "이건 내 거다" 인지하기 🎯 F01

**P1(민지)는** 에타 자유게시판에서 "이거 해봄ㅋㅋㅋ" 글을 보고 호기심에 링크를 눌렀다. **첫 화면을 1초 보고** "어 이거 연기 전공생용이네"가 즉시 인지되어야 **이탈하지 않고 시작 버튼을 누르고 싶다.**

**수용 조건**
- [ ] 첫 화면 위쪽에 "연기인을 위한" 또는 "연영과/배우 전용" 식의 도메인 시그널이 명확히 보임
- [ ] 12~14문항·2~3분 소요라는 분량 정보가 시작 버튼 근처에 있음
- [ ] 시작 버튼(CTA)이 모바일 화면 첫 뷰포트 안에 들어옴
- [ ] 로딩이 2초 이내

**연결 기능**: F01 랜딩 페이지

---

## US-02 — 연기인 맥락이 살아있는 문항 풀기 🎬 F02

**P1(민지)는** 보통 MBTI 진단은 "회사에서 / 사람들 만날 때" 같은 일반 상황 문항인데, **이번엔 "리허설·캐스팅·대본 받음" 같은 진짜 연기 현장 상황**으로 풀어보면서 "오 이건 진짜 연영과용이네" 라는 도메인 신뢰감을 얻고 싶다.

**수용 조건**
- [ ] 모든 문항이 연기/공연/리허설 등 도메인 맥락임 (일반 진단 환원 금지)
- [ ] 1문항 = 4지선다, 각 선택지가 4축 중 한 극(極)에 대응
- [ ] 총 12~14문항
- [ ] 문항 카피의 위트 톤이 idea-brief의 "진지 30 / 재미 70"을 따름

**연결 기능**: F02 시나리오 4지선다 문항

---

## US-03 — "얼마 남았지?" 가 항상 보이기 ⏳ F03

**P1(민지)는** 단톡방 보다가 잠깐 풀고 있어서, **지금 몇 문항 남았는지·끝이 가까운지가 항상 보여야** 도중 이탈하지 않고 끝까지 풀고 싶다.

**수용 조건**
- [ ] 진행바 또는 "3 / 12" 카운터가 모든 문항 화면 상단에 노출
- [ ] 이전 문항으로 돌아가는 버튼이 있음 (또는 의도적으로 없음 — 결정 사항)
- [ ] 선택 즉시 다음 문항으로 자동 진행 (탭 1회로 진행)
- [ ] 마지막 문항에서 "결과 보기" 버튼이 명확히 다름

**연결 기능**: F03 문항 진행 UI

---

## US-04 — 내 답을 4글자 유형 코드로 받기 🧮 F04

**P1(민지)는** 12~14문항 다 풀고 나서, **내 선택이 4축(M/T, I/P, N/A, B/S) 점수로 환산되어 16유형 중 하나로 떨어지는 결과**를 보고 싶다. (계산 로직 자체는 사용자에게 직접 보이진 않지만, 결과가 일관되게 나와야 함)

**수용 조건**
- [ ] 같은 답을 골랐을 때 항상 같은 유형이 나옴 (결정적)
- [ ] 동점 시 어느 극으로 떨어질지 규칙이 명시되어 있음 (tie-break)
- [ ] 유형 분포가 한쪽으로 쏠리지 않도록 문항 가중치가 검토됨
- [ ] 계산 결과는 4글자 코드(예: `MINB`)로 표현됨

**연결 기능**: F04 점수 계산 → 유형 매핑 로직

---

## US-05 — 한 화면에 들어오는 "캡처용" 결과지 📸 F05

**P1(민지)는** 결과가 나온 순간 가장 먼저 **"이거 단톡방에 캡처해서 던질 만한 그림인가"** 를 본능적으로 판단한다. 모바일 한 화면 안에 코드·네이밍·캐치프레이즈·캐릭터 묘사가 다 들어와야 **스크린샷 한 방으로 친구한테 던지고 싶다.**

**수용 조건**
- [ ] 모바일 1뷰포트(약 660px) 안에 4요소가 모두 들어옴: ① 4글자 코드 ② 위트 네이밍 ③ 한 줄 캐치프레이즈 ④ 캐릭터 묘사 3~4줄
- [ ] 캡처 영역이 시각적으로 분리되어 있음 (배경/카드 구분)
- [ ] 16유형 각각의 콘텐츠가 모두 완성되어 있음 (AI 초안 → 본인 폴리싱)
- [ ] 톤 일관성 자체 평가 5점 척도 통과 (idea-brief 리스크 A 대응)

**연결 기능**: F05 결과지 캡처 영역 + 16유형 콘텐츠

---

## US-06 — 스크롤로 더 깊은 "곱씹기" 거리 받기 📜 F06

**P1(민지)는** 캡처는 던졌고, 이제 **혼자 스크롤하면서 "내 유형이 어떤 역할·장르랑 어울린대?", "내 천적·베프 유형이 누구래?"** 를 곱씹어보고 싶다. 이게 "다른 MBTI 진단과 다른 차별점"으로 느껴져야 한다.

**수용 조건**
- [ ] 스크롤 영역에 "🎬 어울리는 역할/장르" 3~4개가 표시됨
- [ ] "⚔️ 천적 유형" 1개 + "🤝 베프 유형" 1개가 표시됨
- [ ] 천적/베프는 다른 16유형 중 1개씩 매핑되어 있음
- [ ] 캡처 영역 ↔ 스크롤 영역이 시각적으로 명확히 구분됨

**연결 기능**: F06 결과지 스크롤 영역 (차별점 핵심)

---

## US-07 — 캡처 안 뜨는 사고 없이 이미지 저장 💾 F07

**P1(민지)는** "결과 이미지 저장" 버튼 하나로 **모바일 갤러리 또는 다운로드 폴더에 결과 이미지가 깔끔하게 저장**되어, 단톡방에 첨부 이미지로 던지고 싶다. iOS Safari · 안드로이드 Chrome 둘 다에서 막힘 없어야 한다.

**수용 조건**
- [ ] "이미지 저장" 또는 "다운로드" 버튼이 결과지에 명확히 있음
- [ ] iOS Safari · 안드로이드 Chrome · 데스크톱 Chrome 3환경에서 동작 확인됨
- [ ] 저장된 이미지에 캡처 영역 4요소가 모두 포함되어 있음
- [ ] 폰트 잘림·이미지 깨짐·여백 어긋남 없음

**연결 기능**: F07 결과 이미지 캡처/다운로드 (html2canvas 등)

---

## US-08 — 링크 하나만 던지면 친구가 내 결과를 그대로 봄 🔗 F08

**P1(민지)는** 단톡방에 이미지 대신 **링크 한 줄만 던져도, 친구가 그 링크를 누르면 내 결과 화면을 그대로 보고**, 그 자리에서 "나도 풀어볼래" 가 자연스럽게 이어지길 원한다.

**수용 조건**
- [ ] 결과 URL에 유형 코드가 인코딩되어 있음 (예: `/result/MINB`)
- [ ] 그 URL로 다시 들어가도 동일 결과 페이지가 렌더링됨
- [ ] 결과 페이지에 "나도 풀어보기" CTA가 명확히 있음
- [ ] URL 길이가 카톡 1줄 메시지에 자연스럽게 들어감

**연결 기능**: F08 결과 URL 공유 (영속화)

---

## US-09 — 카톡으로 한 번에 보내기 💬 F09

**P1(민지)는** 갤러리에서 이미지 다시 꺼내거나 URL 복붙하는 거 없이, **결과 화면의 "카카오톡 공유" 버튼 한 번**으로 친구 톡방에 결과를 던지고 싶다. 카카오톡 공유 미리보기에는 유형 코드/네이밍이 보여야 한다.

**수용 조건**
- [ ] "카카오톡 공유" 버튼이 결과지에 있음 (인스타 스토리/X 공유는 옵션)
- [ ] OG 태그가 유형별로 동적으로 세팅됨 (제목·설명·이미지)
- [ ] 카카오톡 미리보기 카드에 유형 코드 + 네이밍 + 이미지가 표시됨
- [ ] 공유 미리보기 이미지가 16유형 모두 준비되어 있음

**연결 기능**: F09 카카오톡/SNS 공유 버튼

---

## US-10 — "한 번 더 풀어볼래" 부담 없이 🔁 F10

**P1(민지)는** 결과 보고 "어 이거 약간 애매한데?" 또는 친구가 "너 그거 진짜 너 같아?" 했을 때, **결과지에서 바로 "다시 풀기" 버튼 하나로 처음부터 다시 시작**하고 싶다.

**수용 조건**
- [ ] "다시 풀기" 버튼이 결과지 하단(스크롤 영역)에 명확히 있음
- [ ] 누르면 1번 문항부터 다시 시작됨 (이전 답 초기화)
- [ ] 모바일에서 손가락 닿기 좋은 위치/크기 (최소 44px 높이)
- [ ] 의도치 않은 더블탭 방지

**연결 기능**: F10 "다시 풀기" 버튼

---

## US-11 — 모바일에서 손가락만 써도 다 풀림 📱 F11

**P1(민지)는** 사실상 100% 모바일에서 이걸 풀게 된다. **세로 모드 한 손 사용 기준, 가로 스크롤·핀치줌·확대 없이도 모든 문항·결과지가 자연스럽게 풀려야** 한다.

**수용 조건**
- [ ] 360px ~ 430px 폭에서 가로 스크롤 발생하지 않음
- [ ] 터치 타겟 최소 44×44px (Apple HIG 기준)
- [ ] 폰트가 12pt 이상으로 가독성 확보
- [ ] iOS Safari · 안드로이드 Chrome 실기 1대씩 검증

**연결 기능**: F11 모바일 반응형

---

## US-12 — 친구 유형 결과지로 점프하기 ⚔️🤝 F12

**P1(민지)는** 내 결과지 스크롤 영역에서 "내 천적 = `TPAS`" 라고 보고 나면 **"엥 그게 뭔데?" 싶어서 그 유형 카드를 눌러 그 유형 결과지를 바로 보고** 싶다. 그러면 "오 우리 학과 OO는 딱 이거다" 식으로 비교 사슬이 연장된다.

**수용 조건**
- [ ] 천적/베프 유형 카드가 클릭 가능함
- [ ] 클릭하면 해당 유형 결과지(`/result/{코드}`)로 이동
- [ ] 이동한 결과지에도 천적/베프 카드가 있어 사슬이 끊기지 않음
- [ ] 뒤로 가기로 원래 내 결과지에 정확히 돌아옴

**연결 기능**: F12 천적/베프 유형 클릭 → 해당 유형 결과지 보기

## 🎯 Stage 1.3으로 넘기는 것

이 12개 스토리를 **시간 축에 펼친 사용자 흐름(User Flow)** 으로 변환:

- **유입 → 랜딩 → 시작 → 문항 풀이 → 결과 표시 → 공유 → (재진입/사슬 연장)**
- 위 흐름의 분기점·이탈 가능 지점·재진입 루프를 화살표 도식으로 그리기
- 12개 스토리가 어느 단계에서 발화되는지 매핑
