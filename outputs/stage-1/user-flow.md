## 흐름 설계의 핵심 결정 (Stage 1.3에서 결정)

| 결정 항목 | 선택 | 이유 |
|---|---|---|
| 문항 풀이 중 **뒤로가기** | ✅ **있음** (이전 화살표) | 신중하게 풀고 싶은 유저 존중 / 이전 답 보존 |
| 공유 URL **수신자 진입 시** | ✅ 결과지 바로 표시 + "나도 풀어보기" CTA | 자연스러운 확산 사슬 |
| 문항 진행 | 선택 즉시 자동 다음 | 마찰 최소화 (단, 뒤로가기로 수정 가능) |

## 🔀 분기 1 — 공유 (확산 루프 생성)

```
③ 결과 페이지
   │
   ├─ [📷 이미지 저장]  (US-07, F07)
   │     │  · html2canvas 캡처
   │     │  · 갤러리/다운로드 폴더에 저장
   │     ▼
   │  📱 단톡방에 이미지 첨부 → 친구가 "ㅋㅋ뭐야 이거" → 링크 받음 → ①로 돌아옴 (다른 유저)
   │
   ├─ [🔗 링크 복사]  (US-08, F08)
   │     │  · /result/{코드} URL 복사
   │     ▼
   │  📱 단톡방/DM에 URL 붙여넣기 → 친구 클릭 → ③' 친구가 수신자로 진입 (아래 별도 흐름)
   │
   └─ [💬 카톡 공유]  (US-09, F09)
         │  · 카카오 SDK 호출
         │  · OG 카드(유형 코드+네이밍+이미지)
         ▼
      📱 카톡 톡방에 미리보기 카드 노출 → 친구 클릭 → ③' 친구가 수신자로 진입
```

### 분기 1 보조 흐름 — 공유 수신자가 진입

```

             │

        [공유 분기 반복]  ← 확산 사슬 1단계 더 연장
```

## 🔄 분기 2 — 재진입 ("다시 풀기")

```
③ 결과 페이지
   │
   └─ [🔁 다시 풀기]  (US-10, F10)
         │  · 이전 답 초기화
         ▼
      ② 문항 풀이 (1번 문항부터)
         │
         ▼
      ③ 결과 페이지 (새 유형 or 같은 유형)
```

**언제 발생?**
- 결과가 "약간 애매한데?" 싶을 때
- 친구가 "너 그거 진짜 너 같아?" 라고 했을 때
- 단순 호기심 ("다른 답 골랐으면 뭐 나왔지?")

## ⛓️ 분기 3 — 비교 사슬 연장 (천적/베프 점프)

```
③ 결과 페이지 (예: MINB)
   │
   │  스크롤 영역에서 천적/베프 카드 탭
   │   (US-12, F12)
   │
   ├─ ⚔️ 천적 [TPAS] 카드 클릭
   │     ▼
   │  ③″ /result/TPAS  (천적 유형 결과지)
   │     · 캡처 + 스크롤 영역 동일 구조
   │     · 이 페이지에서도 천적/베프 카드 존재
   │     ▼
   │   [사슬 무한 연장 가능: 친구들 유형 떠올리며 클릭 ↔ 클릭]
   │
   └─ 🤝 베프 [MIAS] 카드 클릭
         ▼
      ③″ /result/MIAS  (베프 유형 결과지)
```

**브라우저 뒤로가기**로 원래 내 결과지(③)에 정확히 복귀 가능 (히스토리 유지).

## ⚠️ 이탈 가능 지점 & 대응

| # | 이탈 지점 | 원인 | 대응 |
|---|---|---|---|
| L1 | 랜딩 → 시작 | "이거 나용 아닌듯" / "분량 길어보임" | 도메인 시그널 강화, 2~3분 표시 |
| L2 | 문항 풀이 중 | 지루함 / 문항 톤 일관성 붕괴 | 진행률 가시화, 문항 위트 톤 유지, 12~14개 상한 |
| L3 | 결과지 첫 화면 | "별 거 없네" | 캡처 영역 4요소 임팩트 (네이밍 + 캐치프레이즈) |
| L4 | 공유 단계 | iOS에서 이미지 저장 실패 | iOS Safari 실기 테스트 (US-07 AC) |
| L5 | 공유 수신자 진입 후 | "친구 결과만 보고 끝" | "나도 풀어보기" CTA를 첫 뷰포트 안에 |

## 🎯 핵심 루프 정의 (확산 메커니즘)

```
[유저 A] 풀기 → 결과 → 공유 (이미지/링크/카톡)
                              │
                              ▼
                         [유저 B] 수신 → 친구 결과 봄 → "나도" → 풀기 → 결과 → 공유
                                                                                │
                                                                                ▼
                                                                          [유저 C] ...
```

→ **공유 → 수신 → 본인 풀이 → 다시 공유** 가 끊김 없이 연결되는 것이 가장 중요한 확산 동력.
→ idea-brief 1차 가치(공유·놀이)와 직결 — 모든 화면 설계는 이 루프 마찰을 줄이는 방향.

## 🎯 Stage 1.4로 넘기는 것

흐름에서 등장한 **고유 화면들** 을 화면 구조로 정리:

1. 랜딩 페이지
2. 문항 페이지 (단일 컴포넌트 재활용)
3. 결과 페이지 (캡처 영역 + 스크롤 영역)
4. 결과 페이지 — 공유 수신자 모드 (③과 동일하지만 CTA 강조)
5. (선택) 404 / 잘못된 코드 페이지

각 화면의 **목적 / 핵심 요소 / 액션 / 다음 이동** 을 표로 정리할 예정.
