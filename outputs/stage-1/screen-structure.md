# Screen Structure — 연기 스타일 MBTI

> Stage 1.4 산출물 — 화면 구조 (와이어프레임 없이 표·요소 목록 중심)
> 작성일: 2026-05-14

## 화면 인벤토리

| ID | 화면명 | URL | 목적 | 우선순위 |
|---|---|---|---|---|
| S1 | 랜딩 페이지 | `/` | 도메인 시그널 + 시작 CTA | 🔴 Must |
| S2 | 문항 페이지 | `/quiz` (또는 SPA 내부 상태) | 시나리오 4지선다 풀이 | 🔴 Must |
| S3 | 결과 페이지 (본인) | `/result/{코드}` | 캡처 영역 + 스크롤 영역, 공유·재진입 액션 | 🔴 Must |
| S3' | 결과 페이지 (공유 수신자) | `/result/{코드}` (referrer ≠ self) | 친구 결과 표시 + "나도 풀어보기" 강조 | 🔴 Must |
| S4 | 잘못된 코드 / 404 | `/result/{비정상코드}`, 그 외 | 안내 + 처음으로 이동 | 🟠 Should |

> S3와 S3'는 동일 URL·동일 컴포넌트지만 진입 컨텍스트에 따라 CTA 강조도가 다릅니다 (수신자 모드 = "나도 풀어보기" 강조).

## S1 — 랜딩 페이지 `/`

**목적**: 첫 1초 안에 "연기인용 진단"임을 인지시키고, 시작 버튼까지 도달시키기.

**연결 스토리/기능**: US-01 / F01

| 요소 | 종류 | 내용/제약 |
|---|---|---|
| 헤더 영역 | 텍스트 | 서비스명 (예: "연기 스타일 MBTI") |
| 메인 카피 | 텍스트 | "연기인을 위한" / "연영과·배우 전용" 도메인 시그널 (1줄, 굵게) |
| 서브 카피 | 텍스트 | 진단의 한 줄 설명 (예: "당신은 어떤 연기 인간인가") |
| 분량/시간 표시 | 텍스트 | "12~14문항 · 약 2~3분" |
| 시각 요소 | 이미지/그래픽 | (선택) 16유형 일부 코드 미리보기 또는 일러스트 1개 |
| 시작 CTA | 버튼 | "시작하기" (탭 → S2) |
| 푸터 (선택) | 텍스트 | 만든이 / GitHub / 피드백 채널 |

**액션 → 다음 이동**
- 시작 CTA 탭 → **S2 (문항 1번)**

**AC 핵심**
- 시작 CTA가 모바일 첫 뷰포트 안
- 도메인 시그널이 메인 카피에 명시
- 로딩 2초 이내

## S2 — 문항 페이지 `/quiz`

**목적**: 12~14개 시나리오 문항을 마찰 없이 풀게 한다.

**연결 스토리/기능**: US-02·03·11 / F02·03·11

| 요소 | 종류 | 내용/제약 |
|---|---|---|
| 진행률 표시 | 텍스트 + 진행바 | "3 / 12" 카운터 + 시각 진행바 (상단 고정) |
| 이전 버튼 | 버튼 (←) | 이전 문항으로 이동. 1번 문항에서는 비활성 |
| 시나리오 카피 | 텍스트 | 도메인 문맥(리허설/캐스팅/대본 등) 1~3줄 |
| 질문 | 텍스트 | "당신은?" 또는 "어떻게 반응하나요?" |
| 선택지 1~4 | 카드 버튼 | 각 선택지가 4축 중 한 극(M/T, I/P, N/A, B/S)에 대응. 선택 즉시 다음 문항 자동 진행 |
| (선택) 선택 표시 | 시각 피드백 | 이전 답이 있으면 해당 선택지가 강조됨 (뒤로가기로 돌아왔을 때) |

**상태**
- 현재 문항 인덱스
- 답안 배열 (수정 가능 — 뒤로가기로 변경 허용)

**액션 → 다음 이동**
- 선택지 탭 → 다음 문항 (자동) / 마지막 문항이면 → **점수 계산 → S3**
- ← 버튼 → 이전 문항 (이전 답 보존)

**AC 핵심**
- 360~430px 폭 가로 스크롤 없음
- 터치 타겟 ≥ 44×44px
- 선택 즉시 자동 진행 (탭 1회)
- 1번 문항에서 뒤로가기 비활성 / 디버깅 진입은 차단

## S3 — 결과 페이지 (본인) `/result/{코드}`

**목적**: 캡처 영역으로 공유 가능성 극대화 + 스크롤 영역으로 곱씹기.

**연결 스토리/기능**: US-04·05·06·07·08·09·10·11·12 / F04·05·06·07·08·09·10·12

### 영역 A — 캡처 영역 (모바일 1뷰포트 안)

| 요소 | 종류 | 내용/제약 |
|---|---|---|
| 유형 코드 | 텍스트 (크게) | 예: `MINB` (4글자, 굵게, 강조 색) |
| 위트 네이밍 | 텍스트 | 예: "비 오는 날 뛰쳐나가는 햄릿형" |
| 한 줄 캐치프레이즈 | 텍스트 | SNS 캡션용 위트 한 줄 |
| 캐릭터 묘사 | 텍스트 (3~4줄) | 도메인 위트+공감 |
| 배경/카드 | 시각 | 캡처 시 인식되는 시각적 경계 (색·테두리) |
| 캡처 메타 (선택) | 텍스트 | 서비스명 워터마크 (이미지 저장 시 자동 포함) |

### 영역 B — 스크롤 영역

| 요소 | 종류 | 내용/제약 |
|---|---|---|
| 어울리는 역할/장르 | 리스트 | 3~4개 (예: "광기 어린 안티히어로", "느와르의 조연") |
| 천적 유형 카드 | 카드 (클릭 가능) | ⚔️ + 다른 16유형 1개 (코드 + 네이밍) → S3 이동 |
| 베프 유형 카드 | 카드 (클릭 가능) | 🤝 + 다른 16유형 1개 (코드 + 네이밍) → S3 이동 |
| 이미지 저장 버튼 | 버튼 | 캡처 영역만 이미지로 다운로드 (F07) |
| 링크 복사 버튼 | 버튼 | `/result/{코드}` URL 클립보드 복사 (F08) |
| 카카오톡 공유 버튼 | 버튼 | 카카오 SDK 호출, OG 카드 (F09) |
| 다시 풀기 버튼 | 버튼 | S2 1번 문항으로 이동, 답 초기화 (F10) |

**액션 → 다음 이동**
- 이미지 저장 → 외부 (갤러리/다운로드)
- 링크 복사 → 외부 (클립보드)
- 카톡 공유 → 외부 (카톡 앱/웹)
- 다시 풀기 → S2 (초기화)
- 천적/베프 카드 → **S3 (다른 유형)** — 사슬 연장
- 뒤로가기 → 이전 S3 또는 S1

**AC 핵심**
- 캡처 영역 4요소가 1뷰포트(≤660px)에 모두 들어옴
- 캡처/스크롤 시각 경계 명확
- 천적/베프 카드 클릭 → 정확한 라우팅, 뒤로가기로 복귀 가능
- 이미지 저장이 iOS Safari + 안드로이드 Chrome 둘 다 동작
- OG 태그가 유형별 동적 (16개)

## S3' — 결과 페이지 (공유 수신자 모드)

**기본**: S3와 동일 URL·동일 컴포넌트.
**차이**: 진입 시 강조 요소가 다름.

**연결 스토리/기능**: US-08 / F08

### S3와의 차이점

| 요소 | 본인(S3) | 수신자(S3') |
|---|---|---|
| 안내 텍스트 (선택) | 없음 | "친구가 풀어본 결과예요" 등 1줄 안내 (옵션) |
| 우선 CTA | "다시 풀기" 동급 | **"나도 풀어보기"** 강조 (스크롤 영역 최상단) |
| "나도 풀어보기" 위치 | (없음) | 캡처 영역 바로 아래, 첫 스크롤 안 |
| 그 외 요소 | 동일 | 동일 |

**모드 판별 기준**
- localStorage에 본인 결과 코드가 저장되어 있는지 확인
- 저장된 코드 ≠ URL 코드 → 수신자 모드로 간주
- (대안: referrer 검사 / 별도 쿼리 파라미터 / 단순히 "본인 풀이 기록 없음" → 수신자 모드)

**액션 → 다음 이동**
- "나도 풀어보기" 탭 → **S2 (1번 문항)** → 자기 결과 S3
- 그 외 액션은 S3와 동일

**AC 핵심**
- "나도 풀어보기" CTA가 캡처 영역 직후 첫 스크롤 영역 안
- 본인 풀이 기록이 있으면 자동으로 S3 모드 (수신자 안내 미노출)

## S4 — 404 / 잘못된 코드 페이지 (🟠 Should)

**목적**: 비정상 URL 진입 시 막다른 길 방지.

| 요소 | 종류 | 내용/제약 |
|---|---|---|
| 안내 텍스트 | 텍스트 | "이 유형은 아직 없는 캐릭터예요" 같은 위트 톤 |
| 처음으로 CTA | 버튼 | → S1 |
| 풀어보기 CTA | 버튼 | → S2 |

**트리거**
- `/result/XXXX` 의 코드가 16유형 화이트리스트 밖
- 그 외 정의되지 않은 경로
