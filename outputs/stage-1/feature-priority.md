# Feature Priority — 연기 스타일 MBTI

> Stage 1.1 산출물 — 기능 우선순위 (MoSCoW 분류 + 작업량 추정)
> 작성일: 2026-05-13

## 우선순위 분류 원칙

- 🔴 **Must (v1 필수)** — 이게 없으면 서비스 자체가 안 되거나, idea-brief의 5대 설계 원칙·1차 가치(공유·놀이)가 무너짐
- 🟠 **Should (v1 권장)** — 핵심 가치 강화. v1 안에 들어가면 좋지만, 시간 부족 시 v1.1로 이연 가능
- 🟡 **Could (Stage 2 후보)** — 재미·검증 강화. 1차 출시 후 반응 보고 추가
- ⚫ **Won't (v1 의도적 제외)** — 이번 v1 스코프 밖

## 작업량 추정 단위

- ⏱️ **S** = 0.5일 이내 / **M** = 0.5~2일 / **L** = 2~5일 / **XL** = 5일+

## 🧭 작업 순서 권장안

idea-brief의 정책("AI 초안 → 본인 폴리싱", "16유형 풀 사이즈")에 맞춰 **콘텐츠 ↔ 기술 병행**으로 가는 게 좋아요:

### Phase 1 — 골격 (1주)
1. F01 랜딩 페이지 (간단 카피만)
2. F03 문항 진행 UI (더미 문항 5개로 시작)
3. F04 점수 계산 로직 (임시 매핑)
4. F05 결과지 캡처 영역 (더미 유형 4개로 시작)
5. F11 모바일 반응형 (Phase 1 끝나기 전 적용)

### Phase 2 — 콘텐츠 (1~2주, 가장 무거운 단계)
6. F02 시나리오 12~14문항 작성 (AI 초안 → 폴리싱)
7. F05 콘텐츠 채우기: 16유형 × (네이밍 + 캐치프레이즈 + 묘사)
8. F06 콘텐츠 채우기: 16유형 × (역할/장르 + 천적/베프)

### Phase 3 — 공유 (3~5일)
9. F07 이미지 캡처/다운로드
10. F08 URL 공유 (유형 코드 기반)
11. F09 카카오톡/SNS 공유 버튼
12. F10 다시 풀기 버튼

### Phase 4 — β 테스트 & 폴리싱 (1주)
13. 학과 친구 10명 테스트 → 유형 분포 확인
14. 시나리오 문항 재조정 (필요 시)
15. 톤 일관성 자체 평가 (5점 척도) → 약한 유형 재작업

### Phase 5 — Should/Could 추가
16. F12 천적/베프 라우팅
17. (시간 남으면) F13 도감 페이지

→ **총 예상 기간**: 4~6주 (사이드 작업 기준, 콘텐츠 작성 시간이 변수)
